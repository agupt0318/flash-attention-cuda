flash-attention-cuda / the matrix that never gets built

fwd + bwd kernels fp32 exact, never approximate

When a language model reads n words, it compares every word with every word: an n×n grid of scores. The grid is cheap to compute and brutal to store. This project is a GPU kernel, written from scratch, built on one refusal: the grid never gets written down. Words stream through a tiny on-chip memory, a running summary is carried in registers, and the final answer comes out identical to the long way. The instrument below runs both ways, live, and checks them against each other.

Read it plainly: on short passages the textbook way is actually fastest (1.1 ms at 512 words), and PyTorch's production kernel still edges this one by about 5%. The win lives in the memory column: at 4,096 words the textbook way needs 4.13 GiB of scratch space and this kernel needs none beyond its inputs. Double the passage again and the textbook way runs out of GPU. This kernel just keeps streaming.
