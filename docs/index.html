flash-attention-cuda / interactive demo

fwd + bwd kernels fp32 · d 32/64 no n×n materialization

Attention compares each of n words with every other word, which produces an n×n matrix of scores. Computing the scores is cheap; storing the matrix is the bottleneck. FlashAttention computes the same result without storing it: K and V stream through on-chip memory in tiles while a running max, sum, and output accumulator stay in registers. This repo implements that kernel from scratch in CUDA. The demo below runs both methods on the same data, in this page, and compares the outputs.

At short sequences the naive version is fastest (1.1 ms at N=512), and torch SDPA is about 5% faster than this kernel at every length. The difference is memory: at N=4096 the naive version allocates 4.13 GiB for score matrices, while the flash-family kernels allocate 0.13 GiB, which is the inputs and output. At this batch size, N=8192 naive would exceed the T4's 16 GiB.
